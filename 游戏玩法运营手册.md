# 修仙大世界游戏玩法运营手册

本文档基于当前项目代码状态整理，目标是作为玩法攻略、数值查阅表与运营手册使用。内容覆盖当前已经实现的主要系统，并尽量区分“已生效规则”和“代码中存在但效果有限或未完全展开的字段”。

## 1. 游戏定位与核心循环

当前版本的核心成长循环如下：

1. 登录创建角色，获得基础属性与初始境界。
2. 通过`打坐`恢复灵力。
3. 使用灵力进行`地图刷怪`、`探索`、`秘境`、`炼丹`等行为。
4. 通过`刷怪`、`探索`、`抽奖`、`秘境`获得装备、灵草、丹方残页、灵石、洗练石、强化石、灵宠等资源。
5. 通过`装备`、`灵宠`、`丹药`、`成就`进一步提升角色战斗与资源效率。
6. 修为达到当前境界上限后突破，进入更高重数。

当前已经形成的主线玩法优先级：

- `地图刷怪`：当前最强的日常升级与掉落玩法。
- `打坐`：当前唯一常规回灵方式。
- `探索`：灵草、丹方残页、部分修为与灵石来源。
- `抽奖`：装备与灵宠主要来源。
- `炼丹`：中后期加速器。
- `秘境`：挑战玩法，提供灵石、洗练石、成就推进。
- `成就`：长期累计型资源与倍率奖励系统。

## 4. 修炼系统

## 4.1 打坐

当前版本中，打坐是唯一常规回灵方式。

### 4.1.1 打坐恢复公式

基础回灵速度：

```text
baseRate(level) = 10 * 1.75^realmIndex * (1 + 0.05 * minorIndex)
realmIndex = (level - 1) / 9
minorIndex = (level - 1) % 9
```

最终打坐速度：

```text
finalRate = baseRate(level) * spiritRate * (1 + spiritRateBonus)
```

其中：

- `spiritRate`：角色永久灵力恢复倍率
- `spiritRateBonus`：丹药等临时加成

### 4.1.2 灵力储量上限

```text
spiritCap = baseRate(level) * 43200 * (1 + spiritCapBonus)
```

说明：

- `43200 秒 = 12 小时`
- 打坐离线最多结算`12小时`
- 打坐超出 12 小时后自动结束
- 若当前灵力已经高于恢复上限，则仍保留已有灵力，但无法再通过打坐继续增加

### 4.1.3 打坐状态规则

- 打坐与刷图互斥。
- 刷图开始时，服务端会自动终止打坐。
- 打坐中轮询状态可实时看到：
  - 当前灵力
  - 当前上限
  - 当前恢复速度
  - 本轮累计恢复量
- 灵力恢复满后，打坐自动结束。

## 4.2 地图刷怪

地图刷怪是当前主升级玩法，定位是：

- 收益高于纯打坐
- 同时产出少量装备、灵草
- 角色死亡后自动复活继续刷
- 离线最多推进`12小时`

### 4.2.3 战斗结算与难度规则

- 每`1秒`结算一次当前战斗进度。
- 若本轮失败，角色不会退出地图，而是进入复活倒计时。
- 复活时间基础规则：

```text
baseSeconds = 6 + mapMinLevel / 8
reduceSeconds = (playerLevel - mapMinLevel) / 15
最终范围限制在 3 秒 ~ 30 秒
```

- 前五张地图（最低等级`<= 40`）怪物强度按固定`killCount`计算，不随累计击杀持续抬高。
- 后四张地图保持动态成长，实际击杀越多，怪物越强。

### 4.2.4 怪物品质与收益倍率

怪物分三种：

| 品质 | 基础出现逻辑 | 修为倍率 | 掉落倍率 |
|---|---|---:|---:|
| 普通 | 默认 | 1.0 | 1.0 |
| 精英 | `0.06 + min(0.08, killCount * 0.0008)` | 1.35 | 3.0 |
| 首领 | `0.005 + min(0.03, killCount * 0.0005)` | 1.8 | 8.0 |

### 4.2.5 刷图中的额外收益来源

- 福缘双倍修为：

```text
触发概率 = 0.3 * luck
```

- 修炼倍率：
  - 最终修为还会再乘`cultivationRate`
- 自动回血：
  - 默认基础比例 `10%`
  - 上限比例 `45%`
  - 可被涅槃丹等效果继续提高
- 击杀返灵：
  - 触发概率：`20%`
  - 返还比例：`10% ~ 35%` 的本次灵力消耗
- 装备掉落：

```text
基础掉率 = (0.002 + RewardFactor * 0.0015) * 品质掉落倍率
最高封顶 = 8%
```

- 灵草掉落：

```text
基础掉率 = (0.0012 + RewardFactor * 0.0008) * 品质掉落倍率
最高封顶 = 3%
```

### 5.2 探索基础逻辑

- 探索前先扣除地点固定灵力。
- 若触发事件，则进入事件结算。
- 若未触发事件，则按照地点奖励池抽取奖励类型。
- 探索事件触发概率：

```text
0.3 * luck
```

- 非事件奖励倍率加成：

```text
若 rand < 0.5 * luck，则本次奖励倍率为 1.5，否则为 1
```

说明：

### 8.6 装备强化

- 强化上限：`+100`
- 强化成本：

```text
cost = (当前强化等级 + 1) * 10 强化石
```

- 强化成功率：

```text
successRate = 1 - 当前强化等级 * 0.05
最低可降到 0
```

- 强化成功时：
  - 所有词条提升约`10%`
  - 百分比词条保留两位小数
  - 固定数值词条四舍五入

### 8.7 装备洗练

- 单次固定消耗：`10`洗练石
- 洗练逻辑：
  - 每次随机改动`1~3`条词条
  - 每条词条有`30%`概率替换成该部位允许的其他词条
  - 新词条数值范围限制在旧值的`70% ~ 130%`

### 9.4 灵宠升级

- 升级消耗：

```text
cost = 当前等级 * 10 灵宠精魄
```

- 升级后：
  - 等级 `+1`
  - 基础属性按稀有度倍率每级约再涨 `1% * qualityMultiplier`
  - 百分比属性每级增加 `0.01 * qualityMultiplier`

### 9.5 灵宠升星

- 条件：
  - 必须使用`相同稀有度`
  - 必须使用`相同名称`
  - 不能吞自身
- 升星结果：
  - 主宠 `star + 1`
  - 被吞宠物消失
  - 返还被吞宠物一部分升级消耗精魄：

```text
返还精魄 = (被吞宠等级 - 1) * 10
```

说明：

- 当前代码中`star`字段已存在并增长，但没有额外战斗加成逻辑，更多是为后续成长扩展预留。

## 10. 抽奖系统

### 10.1 抽奖类型

- `all`：装备与灵宠混合
- `equipment`：只抽装备
- `pet`：只抽灵宠

### 10.2 抽奖成本

- 普通抽：`100灵石 / 次`
- 开启心愿：`200灵石 / 次`
- 单次最大抽数：`100`

### 10.3 心愿机制

- 可指定目标装备品质或灵宠稀有度。
- 当前实现不是保底，而是直接提高目标档位概率，再按剩余概率重新缩放其他档位。

### 10.4 自动处理

- 自动卖装备：按设定品质直接转强化石
- 自动放生灵宠：按设定稀有度不入背包
- 灵宠精魄会在抽到当下就发放

### 10.5 容量限制

- 当抽奖类型包含灵宠时，若背包内灵宠数量已达`100`只，则不可继续抽灵宠。

## 11. 炼丹系统

炼丹建立在三层基础上：

- 材料层：灵草
- 图纸层：丹方残页解锁后的丹方
- 成功率层：品级成功率

当前版本运营上可以这样理解：

- 探索负责产出“材料 + 残页”
- 炼丹负责把探索成果转为短时爆发
- 中高品级丹药是高价值消耗品

## 12. 秘境系统

### 12.1 秘境难度

当前支持 5 个倍率难度：

- `1`
- `2`
- `5`
- `10`
- `100`

每个倍率记录独立最高层数。

### 12.2 开局逻辑

- 进入秘境后，从该难度个人最高层开始。
- 实际挑战楼层为“当前最高层 + 1”。

### 12.3 增益选择机制

- 第`1`层必选增益
- 此后每逢`5`层也会进入增益选择
- 每次提供`3`个选项
- 刷新次数随机 `1~3`

### 12.4 秘境奖励规则

胜利时：

```text
灵石奖励 = 10 * 楼层 * 难度
```

额外奖励：

- `5`层倍数且非`10`层倍数：精英层，奖励`difficulty`颗洗练石
- `10`层倍数：首领层，记录 Boss 击杀

### 12.5 失败惩罚

- 难度不等于`100`：
  - 随机损失当前修为的`10% ~ 50%`
- 难度等于`100`：
  - 随机掉落`1~3`个境界等级
  - 当前修为清零

### 12.6 敌人强度公式

敌方基础数值随`floor * difficulty`线性抬升：

- 生命：`100 + difficulty * floor * 200`
- 伤害：`8 + difficulty * floor * 2`
- 防御：`3 + difficulty * floor * 2`
- 速度：`5 + difficulty * floor * 2`

百分比属性同样线性增长，并且：

- 精英：基础全属性乘`1.5`
- 首领：基础全属性乘`2.0`

### 12.7 秘境增益池

普通增益示例：

- 气血增加：血量 +10%
- 小幅强化：最终伤害 +10%
- 铁壁：防御 +20%
- 疾风：速度 +15%
- 会心：暴击率 +15%
- 轻身：闪避率 +15%
- 吸血：吸血率 +10%
- 战意：战斗属性 +10%

稀有增益示例：

- 防御大师：防御 +10%
- 会心精通：暴击率 +10%，暴伤 +20%
- 无影：闪避 +10%
- 连击精通：连击 +10%
- 血魔：吸血 +5%
- 震慑：眩晕 +5%

史诗增益示例：

- 极限突破：战斗属性 +50%，最终伤害 +50%
- 天道庇护：最终减伤 +30%
- 战斗大师：战斗属性与抗性 +25%
- 不朽之躯：生命上限 +100%，最终减伤 +20%
- 天人合一：战斗属性 +40%，生命 +50%
- 战圣至尊：暴击率 +40%，暴伤 +80%，最终伤害 +20%

## 13. 成就系统

### 13.1 成就分类

当前共 `9` 大类，每类 `10` 条，共 `90` 条成就：

| 分类ID | 分类名 |
|---|---|
| equipment | 装备成就 |
| dungeon_explore | 秘境探索 |
| dungeon_combat | 秘境战斗 |
| cultivation | 修炼成就 |
| breakthrough | 突破成就 |
| exploration | 探索成就 |
| collection | 收集成就 |
| resources | 资源成就 |
| alchemy | 炼丹成就 |

### 13.2 成就奖励字段

成就奖励会直接作用于角色资源或永久倍率：

| 字段 | 作用 |
|---|---|
| Spirit | 直接增加灵力 |
| SpiritRate | 永久乘算打坐恢复倍率 |
| HerbRate | 永久乘算灵草收益倍率 |
| AlchemyRate | 永久乘算炼丹相关倍率 |
| Luck | 永久乘算福缘 |

### 13.3 当前成就覆盖方向

按系统重点看，成就主要鼓励以下行为：

- 装备：获得数量、极品/仙品、强化等级、完整套装
- 秘境探索：通过层数 5 / 10 / 20 / 30 / 50 / 75 / 100 / 150 / 200
- 秘境战斗：普通怪击杀、精英击杀、Boss 击杀、总击杀
- 修炼：累计修炼时长、境界成长
- 突破：突破次数与高阶境界解锁
- 探索：探索次数、事件次数、资源获取
- 收集：灵草、丹方、装备、灵宠等收藏
- 资源：灵石、灵力、强化石、洗练石等累计持有或消耗
- 炼丹：炼丹次数、高品质炼丹数量

说明：

- 当前成就奖励里已经存在大量永久倍率奖励，尤其是`SpiritRate`、`HerbRate`、`AlchemyRate`与`Luck`。
- 这意味着成就系统不仅是一次性奖励，也是一条长期成长线。

## 14. 当前版本需要注意的实现细节

以下内容是为了避免手册使用时产生误解：

- 打坐已经替代旧版“自动自然回灵”，当前查询角色数据不会再按时间直接补灵力。
- 灵宠`star`字段当前已可增加，但尚未形成独立加成公式。
- 丹药效果字段比较多，当前已经完整接入的重点是：
  - 灵力恢复倍率
  - 灵力上限
  - 回灵丹即时回灵
  - 战斗类增益
- 刷图数值目前已经非常陡峭，高级图基础收益与消耗都很高，这属于当前代码真实状态，不代表已经做完最终平衡。
